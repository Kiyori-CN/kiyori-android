## 命名与去重规则

本包由三部分合并:
1. **原矢量化包 43 个**(来自应用截图第一轮提取)
2. **第二轮截图 58 个**(本次新增,来自 8 张应用截图)
3. **图标库 174 个**(原 browser-icons-v2,已翻译成中文名)

当多个图标功能相近但**视觉样式不同**时,采用限定后缀区分:

- 书签:`书签`(堆叠书)、`书签-标签`(丝带)、`书签-彩色`(蓝圆星)、`书签-简`(线稿)、`书签-加`、`书签-实心`、`书签-方`、`书签-堆叠`
- 刷新:`刷新`(光盘)、`刷新-箭头`、`刷新-简`、`刷新-顺时针`、`刷新-光盘`、`顶栏-刷新`
- 首页:`首页`、`主页`、`首页-简`、`首页-五边形`、`首页-Q`
- 历史:`历史`、`历史-彩色`、`历史-计时器`、`历史-简`、`历史-实心`
- 下载:`下载`、`下载-彩色`、`下载-彩色-圆`、`下载-文件`、`下载-简`、`下载-盒子`、`下载-实心`
- 标签页:`标签页`、`标签页-新建`、`标签页-关闭`、`标签页-多`、`多窗口`
- 箭头/折角:四方向各两套(`箭头-上/下/左/右`、`折角-上/下/左/右`)

**带"-简"后缀**的图标,通常是图标库原版的线稿风格,方便需要简洁外观时选用。
**不带后缀**的图标,通常来自实际应用截图,视觉更具特色。

## 矢量化工艺说明(仅适用于中文部分 101 个图标)

按 `icon-extraction-vectorization` skill v1.0 流水线重建:

1. **像素级定位** — 行列连通块分析,自动识别图标中心与边界
2. **高质量上采样** — Lanczos 放大到 960px 最长边 + 高斯模糊抹平 JPEG 块效应
3. **像素三分类** — 背景 / 暗色线稿 / 彩色,**分层独立处理**
4. **暗色百分位取色** — 取最暗 25% 像素均值 × 0.6,还原纯深黑,避免"黑变灰"
5. **HSV 色相环聚类** — 多色图标(如彩色圆底 + 白色前景)正确分离
6. **结构化小元素保留** — 感叹号的点、时钟的刻度、字母内部不会被当成噪声删除
7. **potrace 位图追踪** — 每个色层生成贝塞尔曲线 path
8. **cairosvg 从 1536px 下采样** — 即使 32×32 小图也有完美抗锯齿

对浅灰线稿和带灰色卡片背景的图标采用了**变体管线**(亮度优先判断背景),确保浅灰元素不被误判为背景。

**图标库 174 个**已是原生 SVG,只做了标准套件生成:每个从 SVG 渲染 6 档 PNG + WebP + 多尺寸 ICO + 白底版本。

## 使用建议

| 用途 | 推荐格式 |
|---|---|
| Web 现代浏览器(首选) | SVG |
| Web 兼容老浏览器 | WebP / PNG |
| Android 资源目录 | 多尺寸 PNG(1024 / 512 / 256 / 128 / 64) |
| iOS App 图标 | 1024×1024 PNG + 自行切 Assets |
| Windows 桌面/任务栏 | ICO(已内含多尺寸) |
| 打印/海报 | SVG → 导出任意 DPI 的 PNG |
| 不透明卡片场景 | `_256_白底.png` |

Android UI 现在需要直接从对应图标目录中取资源:优先使用同名 SVG 作为可编辑源,只有在必须使用位图时再选对应尺寸 PNG。

`软件图标/kiyori-app-icon-1024.png` 不属于上面的 275 个标准图标目录,它只用于应用启动图标生成。

SVG 是唯一的源文件,其他格式都能通过 `cairosvg` 或 Inkscape 等工具从 SVG 重新渲染任意尺寸。
